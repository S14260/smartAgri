# 智慧农业遥感监测与巡田决策平台

## 项目概述

这是一款基于 **Java Spring Boot + Python Flask + 前端可视化** 的智慧农业管理系统，实现了地块管理、遥感影像处理、AI智能巡田、异常检测、路径规划等完整功能。

---

## 系统架构

### 三层分离架构

```
前端可视化层 ──HTTP/SSE──> Java 业务层 ──HTTP──> Python 计算层
 (浏览器)                 (:8080)               (:8000)
```

| 层级 | 技术 | 端口 | 职责 |
|------|------|------|------|
| **前端可视化层** | LayUI + Leaflet + CesiumJS + ECharts | - | 用户界面、地图交互、数据展示 |
| **Java 业务层** | Spring Boot 2.6.13 + JPA + MyBatis-Plus | 8080 | 业务逻辑、用户认证、API网关 |
| **Python 计算层** | Flask + GDAL + Rasterio + NumPy + SciPy | 8000 | 遥感处理、异常检测、路径规划 |

## 核心功能模块

### 1. 地块全生命周期管理

- **双角色体系**：管理员和普通用户各自独立的地块管理
- **地图圈地**：基于 Leaflet.draw 支持多边形（polygon）和圆形（circle）绘制
- **实时面积计算**：绘制时自动计算地块面积（亩）
- **逆地理编码**：高德地图 API 自动填充地块地址
- **完整信息管理**：
  - 地块名称、上季/本季作物
  - 土壤类型（砂质土/黏质土/壤土）
  - 灌溉条件（滴灌/渠灌/自然降水）
  - 土地类型（流转/承包/开垦荒地）
  - 联系人、电话、坐标、面积、地址、地区
- **批量操作**：支持批量删除、按地区筛选

### 2. 多源遥感影像处理

- **光谱指数支持**：NDVI、EVI、SAVI、NDWI 四种指数
- **坐标系处理**：自动检测与重投影（非WGS84自动转换）
- **矢量裁剪**：基于地块边界的精确裁剪
- **超分辨率**：Lanczos 6倍插值提升分辨率
- **配色方案**：13级NDVI配色（深红→深绿）
- **边缘抗锯齿**：基于欧氏距离变换的边缘平滑
- **二次裁边**：去除边界伪影

### 3. 多时相异常检测

#### 子区域分割
- 按NDVI值分为7个等级
- 连通区分析提取空间连续子区域
- 面积过滤：去除 <500像素（噪声）和 >192000像素（过大）

#### 多指标联合判定
| 异常类型 | 判定指标 |
|----------|----------|
| 积水 | NDWI升高 + NDVI降低 |
| 病虫害 | EVI + SAVI + NDVI 综合异常 |
| 干旱 | NDVI持续下降趋势 |

#### 时序异常检测
- Delta NDVI：相邻时相变化量
- Z-Score：统计偏离程度
- 线性回归斜率：长期趋势分析

#### 综合优先级排序
```
priority = 0.65 × severity + 0.25 × area + 0.1 × temporal - 0.05 × edge_penalty
```

### 4. 智能巡田路径规划

#### 异常权重计算
```python
weight = 0.5 × priority + 0.3 × severity + 0.2 × log(area + 1)
```

#### 路径优化算法
- **地块内路径**：最近邻算法 + 最西端点起点优化
- **地块间路径**：异常驱动 + 距离联合决策函数
- **2-opt局部优化**：200次迭代消除路径交叉
- **距离计算**：Haversine球面距离公式

#### 输出结果
- GeoJSON 格式路径数据
- 可视化 PNG 图像
- 量化评估报告

### 5. AI Agent 智能助手（核心亮点）

#### ReAct 模式
基于 **Reasoning + Acting** 的智能决策模式：
1. 用户输入自然语言请求
2. LLM 推理理解意图，规划执行步骤
3. 调用工具获取数据或执行操作
4. 将工具结果回传 LLM 继续推理
5. 输出最终回答或继续调用工具

#### 5个自动注册工具
| 工具名称 | 功能 |
|----------|------|
| `query_plots` | 查询地块信息 |
| `trigger_anomaly_detection` | 启动异常检测任务 |
| `get_anomaly_records` | 获取异常记录（列表/详情） |
| `explain_anomaly` | 异常智能解释分析 |
| `generate_patrol_route` | 生成巡田路径规划 |

#### SSE 流式输出
- `step`：执行步骤
- `thought`：思考过程
- `tool_start`：工具调用开始
- `tool_result`：工具执行结果
- `token`：文本输出
- `done`：完成

#### 对话管理
- ConcurrentHashMap 维护多轮对话历史
- 最多保留30条历史消息
- 支持清除会话历史

### 6. 数据驾驶舱

#### 统计卡片
- 地块总数
- 总亩数
- 异常地块数
- 巡田工单数

#### ECharts 图表
- 作物分布饼图
- 地区分布柱状图
- 土壤类型统计

### 7. GPS 打卡签到

- **多边形判断**：基于射线法的内外判断
- **圆形判断**：基于 Haversine 公式的距离计算
- **自动匹配**：遍历所有地块进行位置匹配
- **记录保存**：打卡时间、坐标、关联地块、备注

## 项目亮点

1. **三层分离架构** - 前端、业务、计算职责清晰，独立扩展
2. **ReAct Agent** - 自然语言驱动的智能巡田，支持多轮对话
3. **多时相遥感融合** - 像素级+区域级变化检测，综合异常判定
4. **2-opt路径优化** - 巡田路径效率提升，消除路径交叉
5. **Cesium 3D可视化** - 沉浸式地球体验，支持NDVI热力图叠加
6. **双Token认证** - 管理端/用户端安全隔离
7. **SSE流式输出** - 实时展示AI思考过程和工具调用状态

## 部署与运行

### 环境要求

- JDK 1.8+
- Maven 3.6+
- MySQL 5.7+（推荐 8.x）
- Python 3.8+

### 部署步骤

#### 1. 数据库初始化
```bash
mysql -u root -p -e "CREATE DATABASE mydb;"
mysql -u root -p mydb < mydb.sql
```

#### 2. 启动 Java 后端
```bash
./mvnw spring-boot:run
# 或打包后运行
mvn package -DskipTests
java -jar target/smart-agriculture-management-0.0.1-SNAPSHOT.jar
```
服务运行在 `http://localhost:8080`

#### 3. 启动 Python Flask 微服务
```bash
cd api
pip install flask flask-cors requests rasterio shapely geopandas numpy scipy matplotlib
python app.py
```
服务运行在 `http://localhost:8000`

#### 4. 访问系统
- 地址：`http://localhost:8080/`
- 默认管理员：`admin` / `admin`
- 默认用户：`zhangsan` / `123456`

## 已知限制

- Agent 会话存储在内存中，服务重启后丢失
- 地块分页为内存分页，大数据量场景需优化
- Python Flask 单线程处理，高并发场景需部署 Gunicorn
